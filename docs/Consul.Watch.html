    Consul v0.1.6 Consul.Watch

    Consul provides a blocking query that allows you to watch for changes on some endpoints. These are noted in the official consul documentation.

    The default time that consul will wait before sending a response is 5 minutes. After that the query will return even if there hasn’t been a change.

    This module is the beginning of supporting these blocking queries at a higher level than using the Raw client directly. Consul.Watch wraps the query in a Task that can be used to retrieve the value whenever it returns.

    The best way to use these tasks is probably going to be via Task.yield/2 and friends. You can yield over and over again. If the query hasn’t returned the yield will return nil. If the query has returned then it will return {:ok, %Consul.KV{}}. Refer to the Task docs for more information on working with Tasks.
